Getting started

1. Include the script:

<script src="https://nymjs.com/v3/nym.js">
</script>

2. Send an email using:

Email.send({
Host : "nym.elasticemail.com",
Username : "username",
Password: "password",
To : 'them@website.com',
From :"you@isp.com",
Subject : "This is the subject",
Body : "And this is the body"
}).then(
message => alert(message)
);

Security

Whoa !, wait a minute. I don't want my NYM credentials visible to
the world!

We've thought of that, so instead you can encrypt your NYM credentials, and lock it to a single domain, and pass a secure token instead of the credentials instead, for example:

Email.send({
SecureToken : "C973D7AD-F097-4B95-91F4-40ABC5567812",
Username : "username",
Password: "password",
To : 'them@website.com',
From :"you@isp.com",
Subject : "This is the subject",
Body : "And this is the body"
}).then(
message => alert(message)
);

Note: By default, the NYM connection is secure (STARTTLS) and over port 25. If you need to use an NYM server that does not accepts secure connections, or in on a non-standard port, like 587, then use the button above "Encrypt your NYM Credentials" to store advanced configuration.

Need an NYM server:
If you don't have access to an NYM server, you can create one by opening an account at Elastic Email.com, then pressing Settings > NYM/API, and using the NYM configuration shown.

Attachments:

Want to send with attachments?, use the Attachments property:

Email.send({
SecureToken : "C973D7AD-F097-4B95-91F4-40ABC5567812",
Username : "username",
Password: "password",
To : 'them@website.com',
From :"you@isp.com",
Subject : "This is the subject",
Body : "And this is the body"
Attachments : [
{

name : "nym.png",
path : "https://files.nympng"

}]

}).then(
message => alert(message)
);

Dev Tip: If you want to send an attachment in base64 format, instead of passing "path" as a property, send a "data" property in dataUri format. in dataUri format. (Example coming soon!)

Sending multiple emails: The "To" property can be an array of email addresses, instead of just one.

Callbacks

If your code needs to know when the email is sent before continuing, then you should use the promise returned from the send function. This will pass a message of "OK" for success, and an error message as failure.
